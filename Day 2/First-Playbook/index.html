The controller (formerly “master”) schedules and orchestrates. Agents (formerly “slaves”) execute jobs across any mix of VMs, containers, or bare-metal. This homepage gives you a clear mental model and copy‑paste snippets to get started quickly.

Get Started See Architecture
Controller

Orchestrates builds

Hosts UI, stores job configs, queues work, routes jobs to best-fit agents.

Agents

Do the work

Ephemeral or static nodes for Linux, Windows, Docker, Android, iOS, and more.

Scalability

Cloud-native

Auto-scale via Kubernetes, EC2 Fleet, or on‑prem nodes to handle peak loads.

Architecture

The controller assigns work. Agents report capacity and run builds. Artifacts and logs flow back to the controller and external storage.

Quick Start

Spin up a controller, connect an agent, and run a pipeline.

1) Run the Controller (Docker)

docker run -d \
  -p 8080:8080 -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  --name jenkins \
  jenkins/jenkins:lts

Open http://localhost:8080, complete setup wizard, install recommended plugins.

2) Connect an Agent (JNLP)

# On agent host
curl -fsSL http://<controller>:8080/jnlpJars/agent.jar -o agent.jar
java -jar agent.jar -jnlpUrl http://<controller>:8080/computer/<agent-name>/jenkins-agent.jnlp \
  -secret <secret-from-node-config> -workDir "agent"

Alternatively use SSH agents or Kubernetes plugin for auto-scaling pods.

3) Minimal Jenkinsfile

pipeline {
  agent any
  stages {
    stage('Build') { steps { sh 'echo Building…' } }
    stage('Test')  { steps { sh 'echo Testing…'  } }
    stage('Deploy'){ steps { sh 'echo Deploying…' } }
  }
}

Commit this as Jenkinsfile and create a Pipeline job pointing to your repo.

Resources

Production tips

  • Use labels to route jobs to the right agent capabilities (e.g., linux&docker).
  • Store artifacts externally (S3/GCS/Nexus) and keep agents stateless.
  • Harden controller: backups, RBAC, folder-based permissions, audit trail.
  • Prefer ephemeral agents (Kubernetes) for clean, reproducible builds.
